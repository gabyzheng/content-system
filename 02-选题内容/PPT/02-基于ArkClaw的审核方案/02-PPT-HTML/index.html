单据审核管理 · 处理流

3.2a
定时拉取待审核清单
Cron 每 30 分钟 → GET BPM /api/tasks/pending → 去重已处理任务
⏰ Cron 触发→ 调 BPM API→ 去重对比
⚠ 401/403 → 提示授权 ⚠ 空清单 → 跳过
3.2b
逐条审核处理（7 步串行）
从前往后逐条处理，避免并发冲突
b1
获取表单明细
GET /api/tasks/{id}/detail → 表单字段 + 附件列表
⚠ 接口失败 → 跳过本条
b2
附件 OCR 处理
下载附件 → 判断类型 → OCR 解析 → 记录状态标识
解析状态标识
● 完整解析（置信度 ≥80%） ● 部分解析（50-80%） ● 无法解析（损坏/加密/低质）
⚠ 下载失败 / OCR 超时 / 格式不支持 → 标记降级
b3
获取审批规则
读 Sheet1 总览 → 匹配流程名 → 解析 JSON（仅取「有效」状态规则）
⚠ 未配置 / JSON 异常 → 跳过审核
b4
AI 审核建议
组装 Prompt（表单数据 + OCR 结果 + 规则 JSON）→ 大模型逐条匹配 → 输出命中结果 + 综合建议
大模型输出
每条规则命中结果 + 理由 · 综合建议（通过/驳回/人工复核）· 风险点汇总
⚠ AI 超时/异常 → 降级（仅展示表单摘要）
b5
更新 BPM 审核建议
PATCH /api/tasks/{id}/review → { suggestion, reasons, risk_level }
⚠ 失败 → 仍推送卡片（标注"BPM 同步失败"）
b6
推送飞书消息卡片
组装交互卡片（单据信息 + AI 建议 + 三按钮）→ 发送至用户私聊
⚠ 推送失败 → 重试 1 次
b7
等待用户操作 → 回调 BPM
审批通过 / 驳回 → PATCH BPM 状态 → 更新卡片 → 继续下一条
用户点击→ 调 BPM 接口→ 更新卡片→ 下一条
⚠ 回调失败 → 不自动重试（避免重复操作）
规则管理（配置→有效性→校验→写回）+ 单据审核（拉取→OCR→规则→AI→推送→回调）· 全链路 17 种异常覆盖

审批规则 — 填写标准与 Sample 示例

1

填写标准（5 字段 × 6 维度）

规则分类说明
说明 标识规则属于哪种校验类型
规范 三选一，不可为空
可选值 表单校验 附件校验 交叉校验
✅ 正确 表单校验 ❌ 错误 写成
细分场景
说明 描述规则适用的具体业务场景
规范 格式：「当…时」。具体可判断，避免模糊
可选值 自由文本
✅ 正确 当报销金额超过5000元时
❌ 错误 当金额比较大时
规则详细描述
说明 规则的完整判断逻辑
规范 字段名 + 运算符 + 阈值。交叉校验写 AND/OR 关系
可选值 自由文本（建议含字段名+运算符+阈值）
✅ 正确 报销金额 > 5000 AND 发票类型 ≠ 专票
❌ 错误 金额超了要检查
执行动作
说明 规则命中后的处理方式
规范 二选一。驳回=直接拒绝；人工复核=标记需人工确认
可选值 驳回 人工复核
✅ 正确 人工复核 ❌ 错误 建议驳回
规则状态
说明 规则的生效状态
规范 新增默认「待验证」→ 校验通过改「有效」→ 废弃改「失效」
可选值 有效 失效 待验证
✅ 正确 待验证 ❌ 错误 生效
