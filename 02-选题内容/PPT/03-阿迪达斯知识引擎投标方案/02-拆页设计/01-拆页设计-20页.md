## P02 · 核心主张

**布局**：big-stat / quote

> "EC 团队不缺知识，缺的是让知识「找得到、用得对、产得快、比得准」的能力。"

**五个关键数字**：
- USP 文案自动化率 **0%** → 目标分钟级

**底部一句话**：知识引擎 = 统一知识底座 + 质量保障 + AI 智能应用 + 自动对比

## P04 · 痛点 1 详解：产品资料分散

**布局**：content-2col
**标题**：痛点 1：产品资料分散，缺少统一数据源
**副标题**：信息不是不够，是找不到、对不齐、用不上

**左栏**：现状描述
- 各 Team 关键产品信息分散在邮件、本地文档、各系统中
- 商品检测报告无快速查询窗口，靠单文件存储查看
- 源头信息无法完全对齐，功能词表靠人工逐一排查
- 系统清单：NOVA、SHAREPOINT、ASSETS HUB、PID、ADIFOX、PLATFORM

**右栏**：根因分析
- 无统一知识入口
- 知识未结构化
- 缺乏自动化检索能力
- 信息传递靠人，效率瓶颈

## P05 · 痛点 2 详解：主档信息检查繁琐

**布局**：content-2col
**标题**：痛点 2：主档信息检查繁琐
**副标题**：一个字段的错误，全平台可见

**左栏**：现状描述
- Model Name、Division、Brand-OC、Category Update、Description in Chinese、Colour-OC、Colour in Chinese、Local Product Type、Gender Update、Material_CN、材质-鞋面等 10+ 字段
- 人工手动创建 + 逐条校对

**右栏**：影响分析
- 流程标准化程度极低
- 人工审核极易错漏
- 错误扩散范围大、修复成本高
- 制约上新速度和数据质量

## P06 · 痛点 3 详解：USP 规则复杂

- 合规要求未系统化
- 完全依赖个人经验

## P08 · 痛点 5 详解：商品对比依赖人工

**布局**：content-2col
**标题**：痛点 5：商品对比依赖人工，质检报告分析耗时
**副标题**：不是没有数据，是数据锁在 PDF 里、散在不同检测方向里

**左栏**：现状描述
- 完全由人工进行制作，无任何自动化辅助
- 人工根据产品属性 → 查阅各产品质检报告 → 逐个数据计算功能参数提升是否符合描述
- 质检报告以完整 PDF 格式存储在 adifox 系统
- 每份报告检测方向不同，能提炼、对比的功能点各不相同

**右栏**：根因分析
- PDF 未结构化：关键数据锁在文档中，无法直接查询和对比
- 检测方向不统一：缺乏标准化的功能参数指标体系
- 对比逻辑靠人脑：无规则引擎或自动校验机制
- 数据孤岛：质检报告与商品主档割裂
- 缺乏知识复用：历史对比结果不沉淀，同类产品反复做重复劳动

## P10 · 能力详解 1：知识入库 & 管理

**布局**：content-2col
**标题**：从分散到统一——知识入库与管理

**左栏**：知识入库
- 多源接入：邮件、文档、业务系统、检测报告
- 格式兼容：PDF/Word/Excel/图片/数据库
- 批量导入：支持历史数据一次性迁移
- API 对接：与现有系统无缝集成

**右栏**：知识管理
- 分类打标：按品类/季节/功能多维度分类
- 标签体系：自动提取功能词、属性词
- 质量检测：错别字/重复/截断/字段校验
- 统一检索：关键词 + 语义 + 条件筛选

**对应痛点**：痛点 1（资料分散）+ 痛点 2（主档校验）

## P11 · 能力详解 2：质量保障 & 规则引擎

**布局**：content-2col
**标题**：从人工到自动——质量保障与规则引擎

**左栏**：质量检测能力
- 错别字检测：入库前自动扫描
- 重复检测：避免知识冗余

- 错误拦截在入库前

**对应痛点**：痛点 2（主档校验）
**关键数字**：10+ 字段校验规则，一次配置持续生效

## P12 · 能力详解 3：AI 智能生成

**布局**：content-2col
**标题**：从人工到智能——AI 驱动的内容生成

**左栏**：USP 智能生成
- 品类卖点库结构化
- 必填维度约束（材质/功能/设计/场景）

**右栏**：PEP 智能生成
- 模板 + 主档 + USP + 素材统一接入
- AI 自动组装详情页
- 按品类/季节/活动批量生成
- 模板统一管理、一键更新

**对应痛点**：痛点 3（USP）+ 痛点 4（PEP）
**关键数字**：USP 0% → 分钟级 | PEP <10% → 70%+

## P13 · 能力详解 4：智能解析 & 自动对比

**布局**：content-2col
**标题**：从人算到机算——智能解析与自动对比

**左栏**：智能解析
- PDF 质检报告入库自动解析
- 提取功能参数、检测值、标准值为结构化字段
- 按检测方向（物理性能/化学成分/安全标准）自动分类
- 建立统一的功能参数标签体系

**右栏**：自动对比
- 将"功能参数提升是否符合描述"的对比逻辑规则化
- 系统自动计算并标记差异
- 质检报告与商品主档通过 SKU/产品 ID 关联
- 历史对比结果自动归档，同类产品复用对比模板

**对应痛点**：痛点 5（商品对比）
**关键数字**：从 100% 人工逐条计算 → 系统自动提取 + 规则对比

## P14 · 能力详解 5：知识运营 SOP

**布局**：content-2col
**标题**：从一次性到可持续——知识运营 SOP

**左栏**：运营能力
- 版本日志：每次变更可追溯
- 有效期配置：过期知识自动下线
- 生命周期管理：创建→发布→更新→归档
- 周期更新提醒：避免知识过期

**右栏**：运营价值
- 知识质量持续保障
- 错误可快速回滚
- 合规审计可追溯
- 知识资产持续沉淀

**核心主张**：知识不是一次性导入，而是持续运营的资产。

## P18 · 差异化优势

**布局**：对比表格
**标题**：知识引擎 vs. 传统知识管理方案

| 维度 | 传统方案 | 知识引擎 |
|------|---------|---------|
| 知识组织 | 文件夹层级，靠人分类 | 多维分类 + 标签体系 + 语义检索 |
| 质量保障 | 无 / 人工抽查 | 入库自动质检（错别字/重复/截断/字段校验） |
| 知识应用 | 人找文档 → 人读 → 人写 | API 检索 → AI 理解 → AI 生成 |
| 非结构化处理 | PDF 靠人读 | 智能解析 → 结构化提取 → 自动对比 |
| 运营管理 | 无生命周期管理 | 版本/有效期/生命周期/周期更新 |
| 系统关系 | 替代现有系统 | 不替代，做统一知识入口层 |
| 扩展性 | 单点使用 | EC → 产品链 → 零售链 → 品牌全域 |
