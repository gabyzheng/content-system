用户体验流

来伊份智能审批系统 — 用户视角全链路

审批规则管理 + 单据审核管理 · 两条主线，一个入口
1

审批规则管理

1a · 查询规则清单
💬
自然语言发起查询
「查看我的审批规则」「我有哪些流程规则」
📋
返回结构化规则清单
流程名称 + 规则数量 + 更新时间 + 状态
点击流程名 → 跳转对应 Sheet 页查看详细规则
⚠ 未配置 → 提示初始化
1b · 查询 / 编辑具体规则
🔍
查看某流程规则明细
「查看采购申请审批规则」→ 返回规则列表（含分类/场景/状态）
⚠ Sheet 不存在 → 提示未配置
✏️
点击链接跳转飞书表格编辑
「编辑采购申请审批规则」→ 返回电子表格链接
用户在飞书表格中直接修改 → 完成后「刷新规则」同步到 Sheet1 JSON
⚠ 链接失效 → 重新获取
1c · 校验规则有效性
🔬
发起规则校验
「校验采购申请审批规则」→ 读取 Sheet 页规则 + 调 BPM 样例数据
⚠ BPM 无样例 → 无法校验
📊
查看校验结果 + 优化建议
命中/未命中矩阵 + 规则优化建议 + 建议新增规则
✅
用户确认 → 自动更新 / 手动编辑
「确认，帮我更新」→ 写回 Sheet 页 + 更新 Sheet1 JSON
「我自己改」→ 返回链接，用户手动编辑
⚠ 写入失败 → 提示手动更新
2

单据审核管理

2a · 自动轮询 + 推送审核卡片
⏰
ArkClaw 每 30 分钟自动拉取 BPM 待审清单
获取待审核单据 → 逐条 AI 审核 → 推送飞书卡片
⚠ 无权限 → 提示授权
📨
收到飞书交互卡片
卡片包含：单据信息 + AI 审核建议 + 操作按钮
📋 采购申请审批 #PR-2026-0608-003
申请人：李四 | 金额：¥85,000 | 供应商：XX 公司
🤖 AI 审核建议：
· [表单校验] 金额超 5 万 → 人工复核
· [交叉校验] 供应商在合格名录 ✅
· [附件校验] 报价单完整 ✅ · 合同已盖章 ✅
综合建议：建议人工复核（金额超限）
2b · 用户操作与反馈
👆
点击「审批通过」或「驳回」
调 BPM 接口更新状态 → 卡片更新为"已处理"
⚠ 接口失败 → 提示跳转 BPM 手动处理
🔗
点击「跳转 BPM」
打开 BPM 详情页，进行更复杂操作（加签/转审等）
⚠ 链接失效 → 提示手动登录
🔄
操作完成 → 继续下一条
ArkClaw 自动处理下一条待审核单据
所有操作均在飞书私聊窗口完成 · 规则管理 + 单据审核 · 一个对话入口，两条体验主线
