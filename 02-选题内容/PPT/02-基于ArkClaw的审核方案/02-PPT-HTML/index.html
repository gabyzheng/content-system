审批规则管理 · 处理流

3.1a
检查配置
读取 config.json → 验证 rule_spreadsheet.token → 调飞书 API 验证表格存在且可访问
读配置→ 验 Token→ 飞书 API 验证
⚠ 不存在 → 提示初始化 ⚠ 无权限 → 提示检查分享
3.1b
检查规则有效性
读取 Sheet1 总览 → 逐流程检查 Sheet 页存在性、内容非空、JSON 一致性、格式合法性
检查维度
Sheet 页存在 ✓ · 内容非空 ✓ · JSON 一致 ✓ · 格式合法 ✓
⚠ Sheet 缺失 / 为空 / JSON 异常 → 输出有效性报告
3.1c
校验具体流程规则
定位 Sheet 页 → 读取规则行 → 调 BPM 样例数据 → 逐条匹配 → 输出优化建议
匹配引擎（仅处理「有效」状态规则）
表单校验：字段值 vs 条件值（>/ 附件校验：附件存在性 + OCR 解析状态
交叉校验：多字段 AND/OR 组合判断
读规则→ 调 BPM 样例→ 逐条匹配→ 优化建议→ 用户确认→ 写回 Sheet + JSON
⚠ BPM 无样例 → 无法校验 ⚠ 写入失败 → 提示手动
⚠ 规则管理异常速查（9 种）
异常场景处理策略
未配置规则表提示初始化
规则表无权限提示检查分享设置
Sheet 页不存在提示规则不完整
Sheet 页为空提示未配置规则
JSON 格式异常建议重新校验更新
BPM 无样例数据提示无法校验
BPM 接口超时重试 1 次 → 提示稍后
规则写入失败提示具体原因
用户确认超时结果过期，重新校验
2

单据审核管理 · 处理流

3.2a
定时拉取待审核清单
Cron 每 30 分钟 → GET BPM /api/tasks/pending → 去重已处理任务
⏰ Cron 触发→ 调 BPM API→ 去重对比
⚠ 401/403 → 提示授权 ⚠ 空清单 → 跳过
3.2b
逐条审核处理（7 步串行）
从前往后逐条处理，避免并发冲突
b1
获取表单明细
GET /api/tasks/{id}/detail → 表单字段 + 附件列表
⚠ 接口失败 → 跳过本条
b2
附件 OCR 处理
下载附件 → 判断类型 → OCR 解析 → 记录状态标识
解析状态标识
● 完整解析（置信度 ≥80%） ● 部分解析（50-80%） ● 无法解析（损坏/加密/低质）
⚠ 下载失败 / OCR 超时 / 格式不支持 → 标记降级
b3
获取审批规则
读 Sheet1 总览 → 匹配流程名 → 解析 JSON（仅取「有效」状态规则）
⚠ 未配置 / JSON 异常 → 跳过审核
b4
AI 审核建议
组装 Prompt（表单数据 + OCR 结果 + 规则 JSON）→ 大模型逐条匹配 → 输出命中结果 + 综合建议
大模型输出
每条规则命中结果 + 理由 · 综合建议（通过/驳回/人工复核）· 风险点汇总
⚠ AI 超时/异常 → 降级（仅展示表单摘要）
b5
更新 BPM 审核建议
PATCH /api/tasks/{id}/review → { suggestion, reasons, risk_level }
⚠ 失败 → 仍推送卡片（标注"BPM 同步失败"）
b6
推送飞书消息卡片
组装交互卡片（单据信息 + AI 建议 + 三按钮）→ 发送至用户私聊
⚠ 推送失败 → 重试 1 次
b7
等待用户操作 → 回调 BPM
审批通过 / 驳回 → PATCH BPM 状态 → 更新卡片 → 继续下一条
用户点击→ 调 BPM 接口→ 更新卡片→ 下一条
⚠ 回调失败 → 不自动重试（避免重复操作）
规则管理（配置→有效性→校验→写回）+ 单据审核（拉取→OCR→规则→AI→推送→回调）· 全链路 17 种异常覆盖
