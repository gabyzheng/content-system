# BCG AI 对焦 — 打透观点

> 基于 BCG Key Questions for meeting preparation 清单，
> 梳理火山引擎 AI 平台（Trae Work + 火山方舟）面向 BCG 的核心论点。

---

## 一、理解 BCG 的提问意图

BCG 这份清单不是泛泛的"了解产品"，而是一套结构化的**供应商评估框架**。
从问题顺序和措辞可以读出几个隐含假设：

1. **BCG 在帮客户选型** — 问题从"市场份额"开始（判断供应商成熟度），而非从"你们有什么功能"开始。这是咨询公司的典型切入方式。
2. **安全是第一过滤条件** — Data Security 模块的问题最具体（训练数据使用、合同承诺、数据驻留、跨境传输），说明安全合规是门槛条件，不通过则后续免谈。
3. **企业集成是差异化战场** — 单独列出 Outlook/SharePoint/Slack 集成 + Agent 能力，说明 BCG 的客户大概率是 Microsoft 生态 + 需要 Agent 编排能力。
4. **商业条款关注灵活性** — 同时问按席位和按用量，说明客户群体多样（大企业要席位制，中小团队要按量）。

## 二、六大模块的核心论点

### Module 1: General — 市场定位

**BCG 想问的**：你们够不够大？服务过哪些像我们客户的公司？和竞品比有什么不同？

**我们的论点**：
- 火山引擎是字节跳动旗下云服务平台，AI 平台承载了字节内部日均万亿级 Token 的推理量
- 服务行业覆盖金融、零售、制造、汽车、医疗等 10+ 行业
- **核心差异化**：不是"一个模型打天下"，而是「开放模型生态 + AI 原生工作台」的双引擎——火山方舟负责模型层，Trae Work 负责应用层

### Module 2: Capability — 模型与 Agent 能力

**BCG 想问的**：模型够不够多？开源模型跟得快不快？Agent 能做到什么程度？

**我们的论点**：
- **模型组合**：豆包（自研）+ DeepSeek-V3/R1 + ChatGLM + MiniMax + 多模态模型，避免单一技术路线绑定
- **开源跟进**：新模型 2-4 周接入，版本化 API 管理，旧版 90 天共存过渡
- **Agent 能力**：Skills（可配置 AI 工作规则）+ MCP 协议（跨系统工具调用）+ Workflows（可视化编排）+ Cron Jobs（定时触发）——四件套完整

### Module 3: Enterprise Integration — 企业集成

**BCG 想问的**：能接入 Microsoft 生态吗？Agent 能力具体支持什么？

**我们的论点**：
- Microsoft 365 / Outlook：OAuth 2.0 集成，邮件理解 + 日历同步
- SharePoint：Microsoft Graph API，文档检索 + 权限体系对接
- Slack：API + Webhook，消息监听 + 智能回复
- 飞书：原生深度集成（文档/消息/日历/审批/多维表格）
- CRM/ERP：开放智能体框架，插件化对接 Salesforce、SAP 等

### Module 4: Data Security — 数据安全

**BCG 想问的**（最尖锐）：客户数据会不会被拿去训练？能给合同承诺吗？数据能留在中国大陆吗？跨境传输怎么管？

**我们的论点**：
- **训练数据**：客户数据不用于模型训练，可提供合同约定的不训练承诺
- **数据驻留**：支持限制在中国大陆，混合云部署（敏感数据本地精调 + 推理弹性调用公有云）
- **安全架构**：三重防护——计算隔离（安全沙箱）+ 存储加密 + 流量审计
- **合规认证**：等保三级、SOC 2（Trae），硬件级 TEE 可信执行环境
- **跨境传输**：支持数据不出境方案，满足 DSL + PIPL 要求

### Module 5: Commercial — 商业模型

**BCG 想问的**：怎么收费？有批量折扣吗？怎么控制成本？

**我们的论点**：
- **双轨定价**：按量计费（Pay-as-you-go，适合探索期）+ 按席位（Per-seat，适合规模化）
- **灵活承诺**：可按月/按年承诺，阶梯折扣
- **成本管控**：可视化用量面板 + 管理员限额 + Auto 智能调度（轻量模型处理常规任务，大模型应对复杂决策）
- **免费额度**：新用户每模型 50 万 Tokens 测试额度

### Module 6: Compliance — 合规

**BCG 想问的**：大中华区以外能用吗？有什么限制？

**我们的论点**：
- 中国大陆：完整部署，数据不出境
- 亚太：混合云/私有化方案
- 全球其他区域：需评估当地合规要求，定制方案
- 三种部署模式：公有云（分钟级）→ 混合云（数据本地+推理云端）→ 私有化（强监管）

---

## 三、关键坦诚点（不应回避的限制）

1. **全球部署覆盖**：中国大陆以外公有云仍在扩展中，部分区域需私有化
2. **非 Microsoft 生态深度集成**：Teams 内直接调用 Agent 仍在路线图中
3. **多语言支持**：核心产品以中英文为主，小语种需定制

## 四、Deck 叙事策略

BCG 是咨询同行，不是终端客户。Deck 的叙事应该：

- **开门见山**：直接按 BCG 的 6 模块框架回应，而非自说自话的产品介绍
- **用 BCG 的语言**：他们问"market share"，我们就答市场覆盖；他们问"competitive advantage"，我们就答差异化
- **安全先行**：Data Security 模块放在商业模型之前，因为这是 BCG 的过滤条件
- **坦诚限制**：不回避当前短板，咨询公司欣赏诚实的供应商
